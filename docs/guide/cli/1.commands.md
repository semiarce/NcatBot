## 初始化项目

```bash
ncatbot init
```

交互式引导会询问机器人 QQ 号和管理员 QQ 号，完成后自动创建：

```python
my-bot/
├── config.yaml          # 配置文件
├── plugins/             # 插件目录
└── plugins/{username}/  # 模板插件
    ├── manifest.toml
    └── plugin.py
```

可用 `--dir` 指定目标目录：`ncatbot init --dir ./my-bot`

## 启动 Bot

### 生产模式

```bash
ncatbot run
```

| 选项 | 说明 |
|------|------|
| `--debug` | 启用调试模式（详细日志输出） |
| `--no-hot-reload` | 禁用插件热重载 |
| `--plugin-dir <路径>` | 指定插件目录（默认 `plugins`） |

### 开发模式

```bash
ncatbot dev
```

等价于 `ncatbot run --debug`，始终开启 debug 和热重载。

## 交互模式（REPL）

直接运行 `ncatbot`（不带子命令）进入交互式 Shell：

```bash
ncatbot [123456789]> config show
ncatbot [123456789]> plugin list
ncatbot [123456789]> exit
```

| 快捷命令 | 说明 |
|----------|------|
| `help` / `h` / `?` | 查看帮助 |
| `exit` / `quit` / `q` | 退出 REPL |

## 插件管理

### 查看已安装插件

```bash
ncatbot plugin list
```

### 创建插件脚手架

```bash
ncatbot plugin create my_plugin
```

自动在 `plugins/` 下生成标准目录结构（`__init__.py`、`manifest.toml`、`plugin.py`、`README.md`）。插件名规则：以字母开头，只能包含字母、数字和下划线。

### 查看 / 启用 / 禁用 / 删除

```bash
ncatbot plugin info my_plugin       # 查看详情
ncatbot plugin enable my_plugin     # 启用
ncatbot plugin disable my_plugin    # 禁用
ncatbot plugin on                   # 全局开启插件加载
ncatbot plugin off                  # 全局关闭插件加载
ncatbot plugin remove my_plugin     # 删除（需确认）
```

## 配置管理

```bash
ncatbot config show                                      # 查看全部配置
ncatbot config get napcat.ws_uri                         # 读取配置项
ncatbot config set napcat.ws_uri "ws://localhost:3001"   # 设置配置项
ncatbot config check                                     # 安全检查
```

值类型自动转换：`true`/`yes` → `bool`，纯数字 → `int`，`[...]` JSON → `list`。

---

## NapCat 诊断

```bash
ncatbot napcat diagnose          # 完整诊断
ncatbot napcat diagnose ws       # WebSocket 连接检测
ncatbot napcat diagnose webui    # WebUI 状态检测
```

诊断命令支持 `--uri` 和 `--token` 选项覆盖配置。
